# GigaChat PPTX Tool

LangChain-инструмент для генерации профессиональных PowerPoint-презентаций из Excel-данных. Работает в связке с GigaChat (или любой другой LangChain-совместимой моделью).

## Что внутри

```
src/pptx_generator/
├── schemas.py       — Pydantic-модели (контракт с LLM)
├── excel_reader.py  — чтение xlsx + профилирование данных
├── prompts.py       — системный промпт для GigaChat (на русском)
├── builder.py       — рендер .pptx (нативные графики, не картинки)
├── tool.py          — LangChain BaseTool
└── __init__.py
```

## Архитектура

```
xlsx → ExcelReader → DataContext ──┐
                                   ├→ GigaChat → JSON → PresentationSpec → PresentationBuilder → .pptx
                  style_prompt ────┘
```

Ключевое решение — **разделение генерации структуры и рендера**:

- LLM возвращает **только JSON** по строгой Pydantic-схеме
- Pydantic ловит галлюцинации и ошибки формата на валидации (с авто-ретраями)
- Билдер детерминированно собирает .pptx, без LLM в этом этапе

Это даёт предсказуемость, тестируемость и возможность кэшировать промежуточные результаты.

## Быстрый старт

```python
from langchain_gigachat import GigaChat
from pptx_generator import GeneratePresentationTool

llm = GigaChat(credentials="...", model="GigaChat-Pro")
tool = GeneratePresentationTool(llm=llm)

result = tool.invoke({
    "xlsx_path": "/path/to/data.xlsx",
    "style_prompt": "Премиальный корпоративный стиль, тёмная палитра",
    "output_path": "/path/to/output.pptx",
})
print(result)  # путь к сгенерированному pptx
```

## Требования к входному xlsx

Два листа с точными именами:

- **`Сырье`** — табличные данные (любая структура: числа, текст, категории)
- **`Суммаризация`** — текст с описанием проблем (любое форматирование)

## Поддерживаемые типы графиков

Все нативные pptx-чарты, редактируемые в самом PowerPoint:

- **bar** — горизонтальные столбцы
- **column** — вертикальные столбцы
- **line** — линейные тренды
- **pie** — круговые диаграммы с автораскраской секторов по палитре
- **scatter** — точечные графики корреляций

## Типы слайдов

| Layout | Назначение |
|---|---|
| `title` | Обложка с декоративной акцентной полосой |
| `section_header` | Разделитель раздела |
| `bullets` | Заголовок + 3-7 буллетов |
| `two_column` | Сравнение в двух колонках с заголовками |
| `chart` | Полноразмерный график |
| `chart_with_text` | График + блок ключевых выводов |
| `kpi` | 2-4 крупных числа в карточках |
| `quote` | Цитата на тёмном фоне с большими кавычками |
| `closing` | Финал |

## Адаптация стиля под промпт

LLM подбирает палитру (6 цветов) и шрифтовую пару под mood, указанный пользователем:

```python
# тёмный корпоративный
"Премиальный отчёт для совета директоров. Палитра — глубокий navy и burgundy."

# яркий стартап
"Презентация для инвесторов SaaS-стартапа. Яркие современные цвета, шрифт без засечек."

# спокойный аналитический
"Технический отчёт. Сдержанная серо-зелёная палитра, моноширинный шрифт для кода."
```

- Чтение xlsx и профилирование колонок

## Production-нюансы

- **Ретраи**: при невалидном JSON tool автоматически отправляет LLM ошибку валидации и просит исправить (по умолчанию до 3 попыток)
- **Безопасность**: tool никогда не выполняет код из ответа LLM — только парсит данные. Билдер не доверяет LLM в рендере
- **Логирование**: все этапы через `logging`, агенту видны структурированные ошибки
- **Структурированный вывод**: для GigaChat-моделей, поддерживающих function calling, рекомендуется использовать `llm.with_structured_output(PresentationSpec)` — это уберёт необходимость в текстовом парсинге

## Расширение

Добавить новый тип слайда:
1. Добавить класс в `schemas.py` (наследник `_BaseSlide`)
2. Добавить в Union `Slide`
3. Реализовать `_render_<name>` в `builder.py`
4. Зарегистрировать в `renderers` dict в `_render_slide`
5. Описать в `prompts.py` для LLM
