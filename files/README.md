# GigaChat PPTX Tool

LangChain-инструмент для генерации профессиональных PowerPoint-презентаций из Excel-данных. Работает в связке с GigaChat (или любой LangChain-совместимой моделью).

## Архитектура

```
xlsx → ExcelReader → DataContext ──┐
                                   ├→ GigaChat → JSON → PresentationSpec → PresentationBuilder → .pptx
                  style_prompt ────┘                          ↑
                                                              │
                                                       (валидация + санитайзеры)
```

Ключевое разделение:

- LLM возвращает только JSON по строгой Pydantic-схеме.
- Pydantic ловит ошибки формата и лишние «рекомендации» до сборки.
- Билдер детерминированно собирает .pptx — LLM в рендере не участвует.

## Что нового в этой версии

**Только анализ, без рекомендаций.**
Промпт явно запрещает императивный язык. Дополнительно работает программная защита в схеме:
- Заголовки слайдов с маркерами «Рекомендации», «План действий», «Что делать», «Следующие шаги» автоматически переименовываются в «Итоги анализа».
- Буллеты с императивами («Следует внедрить...», «Необходимо...», «Рекомендуется...») фильтруются из ответа LLM до рендера.
- Если после фильтрации в слайде не остаётся буллетов — подставляется нейтральный плейсхолдер.

**Защита от переполнения вёрстки.**
- Все тексты идут через `_add_text_box()` с `MSO_AUTO_SIZE.TEXT_TO_FIT_SHAPE`.
- Адаптивный размер шрифта в зависимости от длины (заголовки, KPI, цитаты).
- Жёсткие лимиты длин в Pydantic с авто-обрезанием буллетов и категорий графиков.
- Адаптивная раскладка KPI: 2 элемента — крупные карточки по центру, 3-4 — в ряд.

**Улучшенные графики.**
- Убраны дублирующие заголовки графиков (используется только заголовок слайда).
- На bar/column добавлены подписи значений.
- На line — маркеры на точках + цветные линии.
- На pie — только проценты, без дубля «значение; процент».
- На scatter — крупные кругленькие маркеры в цвете палитры.
- Шрифты осей и легенды берутся из палитры презентации.

## Быстрый старт

```python
from langchain_gigachat import GigaChat
from pptx_generator import GeneratePresentationTool

llm = GigaChat(credentials="...", model="GigaChat-Pro", temperature=0.3)
tool = GeneratePresentationTool(llm=llm)

result = tool.invoke({
    "xlsx_path": "/path/to/data.xlsx",
    "style_prompt": "Премиальный корпоративный стиль, тёмная палитра",
    "output_path": "/path/to/output.pptx",
})
print(result)
```

## Требования к входному xlsx

Два листа с точными именами:

- **`Сырье`** — табличные данные (любая структура: числа, текст, категории).
- **`Суммаризация`** — текст с описанием проблем (любой формат).

## Поддерживаемые типы графиков

Все нативные pptx-чарты (редактируемые в PowerPoint):

- **bar** — горизонтальные столбцы
- **column** — вертикальные столбцы
- **line** — линейные тренды
- **pie** — круговые диаграммы (палитра распространяется на сектора)
- **scatter** — точечные графики

## Типы слайдов

| Layout | Назначение |
|---|---|
| `title` | Обложка с акцентной полосой |
| `section_header` | Разделитель с акцентным маркером |
| `bullets` | Заголовок + 1-6 буллетов (наблюдения) |
| `two_column` | Сравнение в двух колонках |
| `chart` | Полноразмерный график |
| `chart_with_text` | График + блок «Наблюдения» |
| `kpi` | 2-4 крупные метрики (адаптивная раскладка) |
| `quote` | Вынесенный ключевой инсайт из данных |
| `closing` | Финал (визуальный мотив повторяет обложку) |

## Лимиты длин (для LLM и пользователя)

- Заголовок слайда: ≤ 80 символов
- Буллет: ≤ 140 символов (обрезается автоматически)
- KPI value: ≤ 10 символов
- KPI label: ≤ 40 символов
- Категория графика: ≤ 25 символов (обрезается)

Превышение → авто-обрезание, шрифт уменьшается по длине.

## Структура кода

```
src/pptx_generator/
├── schemas.py       — Pydantic-модели + санитайзеры от рекомендаций
├── excel_reader.py  — чтение xlsx + профилирование данных
├── prompts.py       — системный промпт (запрет рекомендаций встроен)
├── builder.py       — рендер .pptx (нативные графики, не картинки)
├── tool.py          — LangChain BaseTool
└── __init__.py
```

## Тестирование

```bash
pytest tests/ -v
```

Покрытие:
- Валидация Pydantic-схем (hex-цвета, длины серий, дискриминация слайдов).
- Чтение xlsx и профилирование колонок.
- Полный smoke-test builder'а (все 9 типов слайдов, все 5 типов графиков).
- Извлечение JSON из LLM-ответа (с обёртками, преамбулами, fenced-блоками).
- **Санитайзер рекомендаций**: переименование заголовков, фильтрация императивных буллетов, плейсхолдер при пустом результате.
- **Лимиты длин**: обрезание длинных буллетов и категорий, отклонение слишком длинных заголовков.

## Production-нюансы

- **Ретраи**: при невалидном JSON tool отправляет LLM ошибку и просит исправить (до 3 попыток).
- **Безопасность**: tool никогда не выполняет код из LLM-ответа.
- **Логирование**: все этапы через `logging`.
- **Стресс-тестовый сэмпл**: `stress_test.py` проверяет вёрстку на максимально длинных текстах.

## Расширение

Добавить новый тип слайда:
1. Класс в `schemas.py` (наследник `_BaseSlide`).
2. Включить в Union `Slide`.
3. Реализовать `_render_<name>` в `builder.py`.
4. Зарегистрировать в `renderers` dict в `_render_slide`.
5. Описать в `prompts.py`.
